### Understanding the interbotix repo
* Uses LMA (Leveberg-Marquadt Algorithm) for ViperX
  * LMA is an iterative numerical IK solver — it starts from the current joint
  configuration and uses gradient descent (with the Levenberg-Marquardt method) to
  iteratively minimize the error between the current end-effector pose and the target
  pose.
  * Comparing pick_ik with LMA and KDL
    * pick_ik vs LMA:
      - LMA is pure gradient descent — it can get stuck in local minima and fail to find a
      solution if the initial joint configuration is far from the target
      - pick_ik adds random restarts to escape local minima, making it much more reliable
      for difficult configurations
      - pick_ik supports custom cost functions (e.g., prefer solutions that minimize joint
      movement, avoid certain configurations) — LMA has no equivalent
      - pick_ik tends to find better solutions for redundant joints

    * pick_ik vs KDL:
      - KDL is the oldest and weakest of the three — slow, frequent failures near
      singularities
      - pick_ik significantly outperforms KDL on both speed and success rate
* NED2's IK solver is configured on the robot's onboard computer, not in the repo
  * Niryo's default ROS stack for the NED2 uses KDL
